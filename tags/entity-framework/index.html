The mystery around DbEntityEntry.State

6/Jul 2016 By Sai Pavan 10 min. read
Tags// C# Entity Framework

While playing around with EF DbContext ChangeTracker API I happened to observe an interesting behavior of DbContext.Entry().State. I will be using AdventureWorks database with Code First from database for this blog post. Initially I have obtained a row using the Find method on the Addresses entity. Now I have called the Entry method on the DbContext to obtain the DbEntityEntry< Address > and accessed the State property on the same. As expected, the State was “Unchanged”.
